03

The teaching trick: shift by one.

For every position, the answer is the next character. One sequence becomes 16 mini-lessons.

Input x · what the model sees · B=16 T=16
Target y · what it should predict at each position · same shape, shifted right by 1
At position 0, the model sees 't' and must predict 'h'. At position 1, it sees 'th' and must predict 'e'. By position 15, it has seen the full window and predicts the next char. One sequence = 16 simultaneous predictions. That's why training is so efficient.
04

Integers become meaning vectors.

An ID is just a label — meaningless. The embedding layer maps each ID to 64 numbers that the model can adjust to encode meaning.

Lookup · each character ID picks a row from a 22×64 embedding table
B×T integers → B×T×C=64 floats
Why this matters: at the start, these 64 numbers per character are random noise. During training, gradient descent nudges them so that similar characters end up with similar vectors. Vowels cluster. Punctuation clusters. The model invents its own semantics.

06

Self-attention. The heart of GPT.

Every token looks at every previous token and asks: "who should I pay attention to right now?" This is where the model learns context.

Step 6a · One vector splits into three roles

Step 6b · Q · Kᵀ produces a T×T scoreboard. The upper triangle is masked off — no peeking at the future.
Each cell (i, j) = how much token i attends to token j.

Bright = strong attention. Hatched cells are causally masked — token #3 cannot peek at token #7's content during training. This is what makes GPT auto-regressive.

After softmax, every row sums to 1.0 — attention weights are a probability distribution over what to look at.
Step 6c · Four heads in parallel — each learns a different conversation pattern
One head might learn "match subjects to verbs," another "track repeated patterns," a third "look at the very recent past." All four outputs concatenate back into a single 64-dim vector. Multi-head attention is parallel pattern-matching.
07

Residual: don't lose what you started with.

The attention output is added back to the input. The original meaning is preserved; attention only enriches it.

Original token
+
Attention output
=
Enriched token
Why residual? When you stack many layers, gradients shrink and the model "forgets" the original input. The x + ... shortcut is a highway — gradients flow back through it directly, and the original signal can never be lost.
LayerNorm then rescales the values so the mean is 0 and variance is 1. Keeps numbers stable across deep stacks — without it, values explode or vanish after a few layers.

The 4× expansion is deliberate. The middle layer is a wide canvas where the model can compute non-linear features (via GELU). Then it compresses the useful features back to 64 dims. Most parameters in any transformer live in these two matrices.
And — same trick — the FFN output is added back to the input via a residual connection. Highway preserved.

Block 1 learns surface patterns: "after 'th' usually comes 'e'."

Block 2 can now reason about block 1's output: "if we're inside the phrase 'the cat sat on', then a noun should come next."

GPT-3's 96 blocks stack reasoning on reasoning until syntax, semantics, and even world knowledge emerge from the same primitive operation: attention + FFN, repeated.
10

The head: project onto the vocabulary.

After all that thinking, the final 64-dim vector gets multiplied by a 64×22 matrix. Out come 22 raw scores — one per character.

B×T×64 @ 64×22 → B×T×22
22 logits — the model's raw "scores" for each possible next character
Logits ≠ probabilities yet. They can be any real number — negative, positive, huge. Just rankings. The actual probabilities come in the next step.
11

Softmax: scores become probabilities.

Two operations: exponentiate (kill negatives, amplify big values), then divide by the sum. Result: 22 numbers that sum to 1.

Same 22 outputs · now probabilities · sums to 1.000
"I'm 87% sure the next character is 'e'."

That single sentence is what the entire 103,552-parameter pipeline produces. Every embedding, every attention score, every FFN multiplication — all of it converges into one probability distribution.
12

Sample. Append. Repeat.

Pick a character from the distribution, glue it to the input, run the whole pipeline again. That's autoregressive generation.

step 1 → input: "the cat sat on "   predicts: 't'
step 2 → input: "the cat sat on t"   predicts: 'h'
step 3 → input: "the cat sat on th"   predicts: 'e'
step 4 → input: "the cat sat on the"   predicts: ' '
step 5 → input: "the cat sat on the "   predicts: 'f'
step 6 → input: "the cat sat on the f"   predicts: 'l'
... and so on, one character at a time
Sliding window: when the input grows beyond 16 chars, we keep only the last 16. The model can't see further back — that's its context window. Real GPTs have windows of 128k+ tokens, but the principle is identical.
13

How the weights got smart.

All 103,552 numbers started random. They were nudged 2,000 times until predictions matched reality.

1
Forward pass

Push a batch through the entire pipeline. Get logits. Get probabilities.

2
Compute loss

Cross-entropy: how surprised was the model by the real answer? Big number = very wrong.

3
Backpropagation

Trace the loss back through every operation. Compute the gradient — how much each weight contributed to the error.

4
Optimizer step (AdamW)

Nudge every weight a tiny bit in the direction that reduces loss. All 103,552 of them.

Training loss · 2000 steps · loss dropped from 3.34 → 0.22
